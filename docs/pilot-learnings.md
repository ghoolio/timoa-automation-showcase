# Pilot Learnings

## Pilot goal

Validate whether a real barber/salon owner can configure and use Timoa with minimal help.

## Early observations

- Mobile dashboard layout needs strong responsive testing.
- Long labels in pill navigation can become cramped on iPhone-sized screens.
- Opening-hours controls need to stack better on small mobile widths.
- Owner onboarding is important because manual setup does not scale.
- Pay-at-salon is the safest first payment mode for pilot usage.
- Real pilot users may provide incorrect account details, so internal provisioning should be easy to correct safely.

## Pilot success criteria

- Owner can log in.
- Owner can configure services.
- Owner can configure opening hours.
- Owner can copy booking link.
- Customer can book from mobile.
- Emails arrive.
- Appointment appears in dashboard.
- Reschedule and cancellation work.
